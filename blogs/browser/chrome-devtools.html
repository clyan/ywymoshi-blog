    Chrome常用调试技巧

    漫思维 浏览器

    将文件下载至本地更清晰, 以上内容学习总结于掘金小册

    # 一. 通用篇

    # 二. Console 篇

    1. console 中的 '$'
    • $: document.querySelector 的别名
    • $0: 获取的是当前鼠标在 element 面板中定位的位置的 dom 节点，默认是body
    • $1...: 之前引用过的 dom 历史
    • $_: 引用刚输出的内容
    • $i
      • 用途： 调试 npm 包
      • 需要安装chrome的 Console Importer 插件 运行$i(npm 包) 如： $i('lodash')
    • console 输出对象类型的内容是地址的引用，修改同一个对象，过个输出的内容都会变
    • console 默认被 async 包裹，可直接使用 await 获取 promise 的值
      • 例： Storage 系统的 占用数 和 空闲数 ： await navigator.storage.estimate()
    • source 面板中 Snippets 右击代码行号
      • 选择 Add conditional breakpoint...(添加条件断点)
      • 输入断点表达式（例如在循环中只需要某次循环断点）
      • 输入 console 表达式输出内容
    • 获取当前 element 面板鼠标停留的 dom 事件
      • monitorEvents, 如 monitorEvents($0, 'click')，$0默认是body, 所以代表的是触发body的点击事件
      • $0可以替换为$(选择器)选择任意节点

    # 三. network 篇

    1. 请求过滤，输入框接受字符串或正则表达式
    2. 表头 initiator 列调用堆栈信息
    3. 自定义请求表: 右击表头选中需要显示的头部
    4. 重新发送 XHR: 右击某个 xhr 请求, 选择Replay xhr
    5. XHR/fetch 断点: Source 面板中 XHR/fetch breakpoint
    6. preserve log 刷新保存之前的请求日志
    7. online 修改网络的类型，如：3G

    # 四. 元素面板篇

    1. 选中element,按快捷键‘h’隐藏element
    2. 选中element ，ctrl + ↑ ，ctrl + ↓ 移动元素
      • Shadow editor,选中shadow样式，调节
      • Cubic bezier(贝塞尔) 编辑器，animation样式调节
    3. 展开所有的子节点
      • 右击节点后的 expand recursively
    4. DOM 断点
      • 点击"..." 符号或者右击你想添加监听的元素
        • subtree modifications
        • 监听任何它内部的节点被 移除 或者 添加的事件
          • attribute modifications
          • 监听选中节点属性变动
        • node removal
          • 监听被选中的元素被 移除 的事件
      • 保持元素的hove，active,等伪类状态
        • 右击element元素，选择force state,选择相应的状态
      • element面板元素事件监听evenet listener

    # 五. Drawer

    Ctrl+p 搜索show drawer,在下方出现的控制台左上角，点击’更多‘符号，显示更多功能

    1. Sensors(传感器): 调试获取位置信息的 API，模拟位置
    2. Network conditions（模拟网络状态）
      • 模拟互联网为典型的3G网络甚至离线！ 这通常用来了解页面资源的大小。或者测试应用的离线功能。 除此之外，Network conditions 面板还可以模拟特定的用户代理
      • 修改user agent（用于判断当前的设备）,模拟手机访问
    3. coverage
      • 跟踪当前加载的 JS 和 CSS 文件的 哪些行正在被执行 ，并显示 未使用字节的百分比 。
      • 绿色 的线条标记 运行 和用 红色 的线条标记 未运行
    4. quick source: 在element面板中打开源码查看
    5. changes: 查看css的已更改的内容 与 最初加载的样式表 进行比较
    6. Render: 监控页面回流重绘
    7. animation: 监听调试页面动画
    8. remote devices: 调试安卓手机
    9. request blocking: 阻止请求某些资源
    10. search: 查询整站所有用到的关键字

    # 六. source

    1. 将本地文件拖入source面板中，选择file System 标签，可在控制台修改代码，并同步系统中的代码，修改样式，立即响应
    2. 代码调用栈call stack 从下往上看
    3. 格式化压缩混淆的代码，点击左下角'{}' 按钮
