    自定义事件统计服务

    漫思维 project

    使用nginx启动一个静态服务器， 存放一个1像素的gif图片，请求这个图片带上参数,

    1. 图片支持跨域
    2. 创建图片发送请求并不需要添加节点到页面
    3. gif图要比其他图片要小
    4. 不会阻塞页面

    # 步骤

    定时拆分日志： 每天的 0:00:00

    1. 获取nginx的 access.log文件
    2. 创建拆分后的日志目录
    3. 按天拆分文件，文件名为昨天的日期
    4. 将access.log的文件拷贝至昨日的文件，
    5. 清空access.log
    6. 结束

    定时分析日志，结果入库： 每天的 4:00:00 (视情况而定)

    1. 获取昨天的日期，找到对应的日期命名的accessLog文件

    2. 使用createReadStream创建流，使用readline逐行读取文件, 避免一次性加载进内存

    3. 获取event.git后面的携带的参数

    4. 按照category， action， label， value 统计

    5. 存储

        {
        'category': { pv: 100, uv: 80 },
        'category.action': { pv: 80, uv: 60 },
        'category.action.label': { pv: 60, uv: 40 },
        'category.action.label.value': { pv: 30, uv: 20 },
    }

    百度统计的实现：

    category： 要监控的目标的类型名称，通常是同一组目标的名字，比如”视频”、”音乐”、”软件”、”游戏”等等。该项必选。

    action： 用户跟目标交互的行为，如”播放”、”暂停”、”下载”等等。该项必选。

    opt_label： 事件的一些额外信息，通常可以是歌曲的名称、软件的名称、链接的名称等等。该项可选。

    opt_value： 事件的一些数值信息，比如权重、时长、价格等等，在报表中可以看到其平均值等数据。该项可选。

    定时删除过期日志： 每天的 4:30:00 (视情况而定)

    1. 遍历日志文件，根据文件名与当前时间的对比，是否超过7天，超过即删除。

    # 定时任务

    方式：

    • linux 的crontab
    • 缺点： 不支持秒级
    • node-cron
      • 优点：支持秒级，使用node服务的形式实现

    定时格式： 0 0 0 * * *： 每天的0：00：00 (视情况而定) 执行 （日志拆分）

    # 文件操作

    文件操作库：fs-extra

    删除文件: removeSync

    创建文件： ensureFileSync

    向文件中写入内容： outputFileSync （会覆盖所有内容）

    清空文件： outputFileSync('filepath', '')

    复制文件内容到另外的文件： copySync

    读取目录： readdirSync
